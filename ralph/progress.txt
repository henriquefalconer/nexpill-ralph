Key porting findings: (1) JS strings are UTF-16/UCS-2 (charCodeAt exposes lone surrogates) but Lean String is UTF-8 with Char = valid scalar value — so ucs2decode/encode and their surrogate fixtures must be modeled over `Array UInt16`, not Lean String. (2) JS `error()` throws RangeError; Lean is pure so decode/encode/toASCII/toUnicode must return `Except PunyError`. (3) Use `Nat` (arbitrary precision) with explicit maxInt guards replicating JS overflow detection. (4) Unbounded `for(;;)` digit/bias loops need explicit termination (fuel or decreasing measure).
