# Public API Surface: `punycode` Object and `module.exports`

**Location:** `punycode.js:416-443`

## Overview

The `punycode` public API is defined as an object literal and exported via CommonJS. This document specifies the object structure, export mechanism, and how the library is consumed by different module systems.

## Object Structure

The `punycode` object literal is defined at `punycode.js:419-441` with the following properties:

### `version` Property

**Location:** `punycode.js:425`

```
'version': '2.3.1'
```

A string representing the current Punycode.js version number. This is documented by JSDoc at `punycode.js:420-424`.

**Important:** This version string must be kept in sync with `package.json:3` (`"version": "2.3.1"`). Synchronization is a matter of developer discipline; there is no automation to prevent them from drifting.

### `ucs2` Namespace

**Location:** `punycode.js:433-436`

```
'ucs2': {
    'decode': ucs2decode,
    'encode': ucs2encode
}
```

An object that exposes UCS-2 conversion methods. This namespace is documented by JSDoc at `punycode.js:426-432`.

- `decode`: Maps to the `ucs2decode` function, which converts JavaScript's internal character representation (UCS-2) to Unicode code points.
- `encode`: Maps to the `ucs2encode` function, which performs the reverse conversion.

These functions are used internally by the top-level `encode` and `decode` methods but are also exposed for direct consumer use.

### Core Conversion Methods

**Locations:** `punycode.js:437-440`

- `'decode': decode` — Decodes Punycode ASCII-compatible encoding to Unicode.
- `'encode': encode` — Encodes Unicode to Punycode ASCII-compatible encoding.
- `'toASCII': toASCII` — Converts a Unicode domain name label to ASCII (Punycode with optional ACE prefix).
- `'toUnicode': toUnicode` — Converts an ASCII domain name label to Unicode.

## Export Mechanism

### CommonJS Export

**Location:** `punycode.js:443`

```
module.exports = punycode;
```

The `punycode` object is exported via CommonJS `module.exports`, making it the default export for Node.js and CommonJS-compatible environments.

### ES Module Equivalent

The ES module version is generated at build time from the CommonJS source. According to `scripts/prepublish.js:6-7`, the build process replaces the line at `punycode.js:443` with:

```
export { ucs2decode, ucs2encode, decode, encode, toASCII, toUnicode };
export default punycode;
```

This transformation preserves both named exports (for tree-shaking) and a default export. The generated file is written to `punycode.es6.js` at `scripts/prepublish.js:17`.

### Package.json Routes

## Non-Obvious Details

1. **Version Duplication:** The `version` field at `punycode.js:425` is a magic string that duplicates the version in `package.json:3`. No automation keeps them synchronized; developers must update both manually.

2. **UCS-2 Exposure:** The `ucs2` namespace exposes `ucs2decode` and `ucs2encode` which are also used internally by the `encode` and `decode` methods. This design allows consumers to work with lower-level Unicode representations if needed.

3. **Build-Time ES Module Generation:** The `punycode.es6.js` file is not hand-written. It is generated by `scripts/prepublish.js` using a regex-based string replacement at `scripts/prepublish.js:6-15`. The build checks for the exact presence of `module.exports = punycode;` and fails if the source has changed in an unexpected way.

4. **Dual Export Strategy:** The ES module version exports both named exports and a default export. This allows consumers to use either `import punycode from 'punycode'` (default) or `import { decode, encode } from 'punycode'` (named), maximizing compatibility and enabling tree-shaking in bundlers.
